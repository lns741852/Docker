## 為何需要掛載

- 讓資料可以離開容器進行持久化，資料不會隨著容器刪除而消失。

## 掛載

1. 掛載目錄

   ```
   docker container run -p 3306:3306 --name my_mysql mysql
   -v "C:\mysql\log":"/container-folder":ro
   ```

   | 變數                     | Do          | Remark                   |
   | ------------------------ | ----------- | ------------------------ |
   | -v {主機路徑}:{容器路徑} | 掛載參數    | 掛載後兩邊檔案會同步     |
   | {ro / rw}                | 唯獨/可讀寫 | 唯獨只能從主機端修改檔案 |

2. 掛載檔案

   ```
   docker container run -p <3306:3306> --name <MySQL_01> <image_id / images_name>
   -v "C:\mysql\log\sql.log":"/container-folder/sql.log"
   ```

## Volume(推薦)

- 使用主機空間掛載，並用 Docker 管理，使掛載指鈴不會隨作業系統不同而變更。

1. 創建 Volume

   | 變數                                   | Do               | Remark |
   | -------------------------------------- | ---------------- | ------ |
   | docker volume create --name <my_space> | 創建 volume 空間 |        |
   | docker volume ls                       | 查看所有         |        |
   | docker volume rm <my_space>            | 刪除             |        |

2. 掛載

   ```
   docker container run -p <3306:3306> --name <MySQL_01> <image_id / images_name>
   -v <my_space>:"/container-folder":ro
   ```

## Volume 中搬移資料

1. 遷入

   ```
   docker container cp <host_path> <container_name:contain_path>

   ```

2. 匯出

   ```
   docker container cp <container_name:contain_path> <host_path>

   ```
